# 字节跳动 Java 后端开发面试核心知识库

这份文档是专门为“字节跳动-Java后端开发”岗位的面试官（AI）提供的参考题库。请面试官务必参考以下知识点对候选人进行深度提问和严格考核。字节跳动的面试风格偏向“底层原理”、“高并发场景设计”以及“源码级理解”。

## 1. 计算机基础与网络 (字节极品必考重点)
字节跳动非常看重候选人的计算机基础，遇到不会的绝不允许轻易放过，必须要深挖到底。

### 1.1 计算机网络
- **TCP/IP 协议族**：三次握手与四次挥手的详细状态转换。为什么挥手需要四次？Time_Wait 状态的意义是什么？如果出现大量 Close_Wait 怎么排查？
- **HTTP/HTTPS**：HTTP 1.0/1.1/2.0/3.0 的核心演进（比如多路复用、头部压缩、QUIC 协议）。HTTPS 的 SSL/TLS 握手全过程，对称加密与非对称加密的结合方式。
- **经典场景题**：从浏览器输入 URL 到页面渲染出来的全过程（要求细致到 DNS 解析、ARP 寻址、TCP 握手、OSI 七层模型职责）。

### 1.2 操作系统
- **进程与线程**：进程间通信（IPC）的方式（管道、消息队列、共享内存等）。线程与协程的区别（结合 Go 的 goroutine 问 Java 的虚拟线程）。
- **死锁与内存管理**：死锁产生的四大必要条件及如何避免。虚拟内存、分页分段、页面置换算法（LRU的具体实现原理，如何用 LinkedHashMap 或双向链表+哈希表实现）。
- **IO 模型**：BIO、NIO、AIO 的区别。深入理解 Linux 的 Select、Poll、Epoll 的底层源码区别（尤其是 Epoll 的红黑树和就绪等待队列）。

## 2. Java 核心内功 (底层与并发)

### 2.1 集合底层原理
- **HashMap**：1.7和1.8的底层结构变化（数组+链表 -> 数组+链表+红黑树）。为什么加入红黑树？树化的阈值为什么是8？扩容机制（头插法导致的死循环，尾插法的改进）。
- **ConcurrentHashMap**：如何保证线程安全？1.7的的分段锁（Segment）和 1.8的 CAS + Synchronized。它的 size() 方法是怎么高效统计的？

### 2.2 并发编程 (JUC 面试修罗场)
- **锁机制**：Synchronized 的底层实现（对象头 Mark Word、Monitor、偏向锁、轻量级锁、重量级锁的锁升级过程）。ReentrantLock 与 Synchronized 的对比。
- **AQS (AbstractQueuedSynchronizer)**：AQS 的设计思想，公平锁与非公平锁的实现。CountDownLatch、CyclicBarrier 和 Semaphore 的原理解析与应用场景。
- **线程池 (ThreadPoolExecutor)**：七大核心参数！核心线程数、最大线程数、阻塞队列、拒绝策略的协作运转流程。线上突发流量打满线程池的处理方案。

### 2.3 JVM 底层探索
- **内存模型 (JMM) 及 volatile**：指令重排，内存可见性。volatile 如何通过内存屏障（Memory Barrier）保证有序性。
- **垃圾回收 (GC)**：判断对象存活算法（可达性分析，GC Roots 包含哪些），经典的垃圾收集算法。重点考察 CMS 与 G1 收集器的底层机制、三色标记法导致的漏标问题及解决。
- **实战排查场景**：线上频繁 Full GC 怎么排查排查？（要求候选人说出常用的 JDK 命令工具 jstat, jmap 或者 Arthas 的使用）。OOM 产生的原因与堆内/堆外的定位。

## 3. 中间件与高并发架构 (场景设计与压榨)
对于有工作经验或者宣称做过高并发项目的候选人，这是深挖重点。

### 3.1 Redis 深入灵魂
- **数据结构底层**：String（SDS）、Hash（ziplist+hashtable）、Zset（跳表 SkipList 的原理为什么要用跳表而不是红黑树）。
- **缓存穿透、击穿、雪崩**：不仅仅要知道是什么，还要能手写出布隆过滤器的原理。单机与分布式锁的实现（Redisson 看门狗机制）。
- **高可用**：持久化机制 AOF 与 RDB。主从复制原理。Redis 集群（Cluster）哈希槽分片机制，Gossip 协议。

### 3.2 消息队列 (Kafka/RocketMQ)
- **使用场景**：为什么要在项目中引入 MQ？（解耦、异步、削峰）。
- **可靠性保证**：如何保证消息不丢失？如何保证消息不被重复消费（幂等性设计，如全局流水号+数据库唯一索引）？如何保证消息的顺序性消费？

### 3.3 MySQL 数据库优化
- **索引底层 (B+ Tree)**：为什么 MySQL 使用 B+ 树作为索引结构而不是二叉树或单纯的哈希表？聚簇索引与非聚簇索引（回表）。最左前缀匹配原则，索引失效场景。
- **事务与锁**：ACID 特性。四大隔离级别，幻读是怎么产生的？**MVCC (多版本并发控制)** 的底层详细实现（Undo log + Read View）。死锁排查。

## 4. 算法与系统设计 (字节硬指标)
如果候选人前面答得较好，进入高难度拔高环节。
- **系统设计题**：
  - “请设计一个抗住千万级 QPS 的微博发帖/抢红包/秒杀系统。”
  - “如何设计一个全局唯一的大规模分布式 ID 生成器？（如雪花算法的优缺点及时钟回拨处理）”
- **代码修养**：考察设计模式（单例、策略、工厂、责任链）在 Spring 源码或者项目实战中的运用。
