# P3 收尾重构 — 2026-04-29

## P3-1：会话存储并发安全

### 问题
- SessionStore 中 `loadUsedAtoms()` → add → `saveUsedAtoms()` 是 read-modify-write
- 同一 recordId 并发 SSE 请求时可能互相覆盖

### 方案
- loadUsedAtoms + saveUsedAtoms 合并为原子操作 `addUsedAtoms(recordId, List<String> newIds)`
- 内部使用 `ConcurrentHashMap.compute()` 或 `synchronized` 块保证原子性

## P3-2：InterviewController DTO 验证

### 问题
- startInterview 接受 `Map<String, Object>`，position 可能为 null，mode 默认值在 Controller 处理
- finishInterview 手动解析 `Long.valueOf(params.get("recordId").toString())` — NPE 风险

### 方案
- 新增 StartInterviewRequest、FinishInterviewRequest DTO
- 用 @Validated + @NotNull/@NotBlank 替代手动校验

## P3-3：EvaluationGenerator 提取

### 问题
- endInterview() 中评估提示词构建 + LLM 调用 + JSON 解析 + 字段映射混在一起

### 方案
- 提取为 EvaluationGenerator 类，接口：`generate(history, wpm, emotionJson) → InterviewRecord`
- 内部封装：提示词格式化 → LLM 调用 → JSON 解析 → 字段填充
