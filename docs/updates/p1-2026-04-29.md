# P1 重构记录 — 2026-04-29

## P1-4：岗位-分类映射外部化

### 问题
- `InterviewServiceImpl.java:307-321` 中岗位到知识库分类的映射硬编码在 if-else 链
- `position.contains("Java")` 会误匹配 "Javascript"
- 新增分类需修改 Java 代码重新部署

### 方案
- 映射关系移至 `application.yml` → `interview.position-categories`（Map<String, List<String>>）
- 新增 `PositionCategoryConfig` 配置类读取映射
- 未匹配岗位抛出明确异常
- `InterviewServiceImpl` 改为注入配置

### 测试
- Java 岗位返回 8 个分类
- 前端岗位返回 2 个分类
- 未匹配岗位抛出 IllegalArgumentException
- "Javascript" 不匹配 Java 规则

## P1-5：持久化逻辑回归服务层

### 问题
- `ResumeController` 直接注入 `ResumeProfileMapper` 做 UPSERT
- `HistoryController` 直接注入 `InterviewRecordMapper` 做查询
- Controller 包含 MyBatis-Plus LambdaQueryWrapper 逻辑

### 方案
- UPSERT 逻辑移入 `ResumeServiceImpl.saveOrUpdateProfile()`
- 历史查询逻辑移入 `InterviewServiceImpl.getHistoryList()`
- Controller 只做参数提取和 Result 封装

### 测试
- ResumeService.saveOrUpdateProfile 使用 Mock Mapper 验证 UPSERT 行为
- InterviewService.getHistoryList 使用 Mock Mapper 验证查询行为
