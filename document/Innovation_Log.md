# AI 面试系统创新与改进日志 (Innovation Log)

本文档记录了项目在演进过程中的核心技术创新与架构优化点。

---

## 知识库原子化重构 (Knowledge Atomization)
**核心价值**: 解决 RAG 系统的“幻觉”与“碎片化”问题。
- **技术突破**: 
    - 摒弃了传统的机械文档切片（Chunking），采用基于 JSON 的**知识原子（Atoms）**架构。
    - 每个原子包含 `Principles`（原理）、`Pitfalls`（面试陷阱）和 `Follow-up`（追问路径）。
- **业务效果**: AI 面试官不再只是模糊地匹配关键词，而是能够精准地捕捉候选人回答中的漏洞（陷阱），并根据预设路径进行“连环炮”式追问，模拟大厂面试的压迫感。

## 多智能体混合面试组 (Multi-Agent Panel)
**核心价值**: 模拟真实多对一面试场景，实现考察维度的专业化解耦。
- **技术突破**:
    - **角色化解耦**: 建立了 `Coordinator`（主考官）、`Technical Lead`（技术官）、`HR BP`（人事专家）三位一体的智能体集群。
    - **动态状态机调度**: 基于对话轮数（Rounds）实现 Agent 自动切换。1轮开场 -> 2-10轮技术压测 -> 11-14轮软技能考察 -> 15轮收尾。
    - **鲁棒性处理**: 针对流式响应中的 Token Usage 解析 Bug，实现了底层的缓冲拼接方案，确保了跨模型驱动的稳定性。
- **业务效果**: 候选人会经历从“温和开场”到“硬核技术对质”再到“职业价值观考察”的完整面试链路，体验感极度接近真实大厂流程。


## 视频面试与多模态情感分析 (Immersive Video & Multimodal Emotion Analysis)
**核心价值**: 补齐非结构化数据（表情、语气、状态）考核短板，打造终极拟真体验。
- **技术突破**:
    - **端侧 AI 情感感知**: 引入 `face-api.js`，通过浏览器前置摄像头在不侵犯隐私（无服务器上传）的前提下，实现每3秒一次的实时情感采样(采样率对cpu要求很高，3秒一次减轻了cpu开销)（Neutral, Happy, Fearful等），计算候选人“自信指数”。
    - **自适应动态压测**: 重构技术面试官内核，实现“难度自适应”且更像真人。AI 会根据候选人上一轮的回答流畅度与准确率，自动决定是深挖底层原理，还是继续问基础概念。
    - **跨模态 TTS 变声系统**: 基于 Web Speech API 构建了 Agent 专属音色库，实现主考官/技术官（沉稳男声）与 HR（亲和女声）的无缝切换，辅以自动静音侦测（VAD）的语音发送(有选择开关)，达成全双工对讲体验。
    - **高维数据降维可视化**: 摒弃传统的复杂折线图，采用定制化 Canvas 热力图（Heatmap），直观呈现候选人多场面试中 A-E 级评价的演进轨迹。
- **业务效果**: 候选人不再是对着屏幕打字的“做题家”，系统能够通过情绪波动和卡壳频率，指出其“表面自信实则发虚”的软肋，逼真度达到线下真实面试的 90% 以上。
